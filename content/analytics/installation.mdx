## The snippet

Paste this into the `<head>` of every page you want to track. Replace `YOUR_TRACKER_ID` with the ID shown in your tracker's **Connect** dialog.

```html
<script
  src="https://beacon.flowtagservices.com/tag.js"
  defer
  async
  data-ft="YOUR_TRACKER_ID"
  data-debug="false"
  data-gtagsync="true"
  data-auto-init="true"
  data-base-url="https://beacon.flowtagservices.com"
></script>
```

## Attributes

| Attribute | Required | Default | Description |
|---|---|---|---|
| `data-ft` | Yes | -- | Your tracker ID. |
| `data-debug` | No | `false` | When `true`, the tag logs debug information to the browser console. |
| `data-gtagsync` | No | `true` | Syncs consent state with Google Tag Manager. Recommended if you use GTM. |
| `data-auto-init` | No | `true` | Automatically starts a session when the page loads. Set to `false` if you want to call `flowtag.init()` manually. |
| `data-base-url` | No | -- | The beacon endpoint. Do not change this value. |

## Framework guides

<Tabs items={['Next.js', 'HTML', 'Shopify', 'WordPress']}>
  <Tab value="Next.js">
    Use the [`@flowtag/next`](/sdk/js/analytics-next) package for a typed React component instead of a raw script tag.

    ```bash
    pnpm add @flowtag/next
    ```

    Add `<Flowtag />` once in your root layout:

    ```tsx app/layout.tsx
    import { Flowtag } from "@flowtag/next";

    export default function RootLayout({ children }: { children: React.ReactNode }) {
      return (
        <html>
          <body>
            {children}
            <Flowtag trackerId="YOUR_TRACKER_ID" />
          </body>
        </html>
      );
    }
    ```

    See the [SDK docs](/sdk/js/analytics-next) for all props and the client API.

          data-ft="YOUR_TRACKER_ID"
          data-debug="false"

## Verifying the tag

After installing the snippet, open the tracker in your Dashboard and click **Connect**. Then click **Check your website** — Flowtag will attempt to detect the tag on your domain.

<Callout>
  If your site dynamically loads scripts, the check may not detect the tag even when it's working correctly. If sessions appear in your dashboard, the tag is active.
</Callout>
